# PgQue benchmarks

## Status

These benchmark notes are **quick'n'dirty preliminary results**.

They are useful for rough direction, not for authoritative claims.

## Current benchmark caveats

- some results come from laptop hardware
- they are not apples-to-apples with every compared system
- they should not be treated as final throughput claims for PgQue
- they need rerun on dedicated server-grade hardware

## How to describe them publicly

Use wording like this:

> Quick'n'dirty preliminary benchmarks. Directional only. Need to be rerun on
> dedicated server-grade hardware with controlled settings and repeated trials.
